# Getting Started

## Install

```bash
npm install -g @carreira-cloud/codebase-wiki
# or run directly:
npx @carreira-cloud/codebase-wiki
```

## Initialize

```bash
cd your-project
codebase-wiki init
```

This auto-detects your workspace (monorepo, single package), creates `.codebase-wiki/config.json`, and prints detected languages, sub-packages, and entry points.

## Start the MCP Server

```bash
codebase-wiki start-mcp
```

This starts a JSON-RPC server on stdio. Add it to your agent's MCP configuration:

```jsonc
{
  "mcp": {
    "codebase-wiki": {
      "type": "local",
      "command": ["bun", "/path/to/codebase-wiki/dist/cli.js", "start-mcp"],
      "environment": {
        "WIKI_DB_PATH": "/path/to/your/project/.codebase-wiki/rag_db"
      },
      "enabled": true
    }
  }
}
```

## Browse the Knowledge Base

```bash
codebase-wiki serve
# → http://localhost:3080
```

The web UI shows:
- **Services tab** — list of all indexed services with documentation
- **Notes tab** — agent self-learning notes with type badges
- **Add Note tab** — manual note creation form
- **Search bar** — full-text search across services and notes
- **Stats dashboard** — service count, content size, notes count

## Generate Documentation

Documentation is generated by AI agents, not by the tool itself. In OpenCode:

```
/wiki generate microservices/oms-microservice
```

The agent will:
1. Explore the service directory using `search_semantic`, `read`, `glob`
2. Collect README, AGENTS.md, source files, config
3. Generate comprehensive Markdown documentation with 9 sections
4. Index it via MCP `wiki_index`

Generate all services at once:
```
/wiki generate-all
```

## Query from CLI

```bash
codebase-wiki search "order checkout"   # Search across services and notes
codebase-wiki get oms-microservice      # Read full documentation
codebase-wiki list                      # List all indexed services
codebase-wiki stats                     # Statistics
```

## Self-Learning

The most powerful feature: agents automatically store discoveries. When an agent finds a pattern, gotcha, or integration detail during any task, it calls `wiki_note`:

```
wiki_note(type="gotcha", topic="Cart TTL hardcoded",
  content="Redis TTL defaults to 86400, not configurable at runtime",
  context="cart-microservice/internal/config/config.go",
  tags="cart,redis,ttl")
```

Six note types: `gotcha`, `pattern`, `integration`, `convention`, `decision`, `tip`.

All notes become searchable. Before starting new work, agents check existing notes for relevant discoveries.
