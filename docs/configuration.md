# Configuration

The codebase-wiki uses a zero-config approach. If no configuration file exists, it auto-detects workspace type, languages, and sub-packages.

## Config File

Generated by `codebase-wiki init` at `.codebase-wiki/config.json`:

```jsonc
{
  "version": 1,
  "project": {
    "name": "my-project",
    "type": "monorepo",
    "root": "."
  },
  "discovery": {
    "include": ["**/*"],
    "exclude": [
      "**/node_modules/**",
      "**/dist/**",
      "**/.git/**",
      "**/*.test.*",
      "**/*.spec.*",
      "**/vendor/**",
      "**/__pycache__/**",
      "**/.venv/**",
      "**/target/**",
      "**/build/**",
      "**/.next/**",
      "**/.turbo/**",
      "**/coverage/**"
    ],
    "maxFileSizeKb": 500
  },
  "vectorStore": {
    "path": "./.codebase-wiki/rag_db",
    "embedding": {
      "provider": "ollama",
      "model": "mxbai-embed-large",
      "baseUrl": "http://127.0.0.1:11434/api",
      "apiKey": null
    }
  },
  "skills": {
    "architecturalPlanning": {
      "enabled": true,
      "minSimilarity": 0.75
    },
    "contextualDebugging": {
      "enabled": true,
      "graphDepth": 3,
      "recentChangesDays": 30
    }
  }
}
```

## Fields

### `project`

| Field | Default | Description |
|-------|---------|-------------|
| `name` | auto | Project name (auto-detected from directory or `package.json`) |
| `type` | auto | `monorepo`, `single-package`, or `auto` |
| `root` | `.` | Root path relative to config file |

### `discovery`

Controls which files are scanned when generating documentation context.

| Field | Default | Description |
|-------|---------|-------------|
| `include` | `["**/*"]` | Glob patterns to include |
| `exclude` | see config | Glob patterns to exclude (node_modules, dist, tests, etc.) |
| `maxFileSizeKb` | 500 | Skip files larger than this |

### `vectorStore`

Storage and embedding configuration.

| Field | Default | Description |
|-------|---------|-------------|
| `path` | `./.codebase-wiki/rag_db` | Storage directory (auto-detects `.opencode/rag_db` if exists) |
| `embedding.provider` | `ollama` | `ollama`, `openai`, or `custom` |
| `embedding.model` | auto | Embedding model name (auto-detected from OpenCodeRAG config) |
| `embedding.baseUrl` | auto | Provider base URL (auto-detected from OpenCodeRAG config) |
| `embedding.apiKey` | auto | API key for OpenAI-compatible providers |

### `skills`

Controls agent skill behavior.

| Field | Default | Description |
|-------|---------|-------------|
| `architecturalPlanning.enabled` | true | Enable pre-planning wiki consultation |
| `architecturalPlanning.minSimilarity` | 0.75 | Minimum similarity threshold for matching docs |
| `contextualDebugging.enabled` | true | Enable pre-debugging wiki consultation |
| `contextualDebugging.graphDepth` | 3 | How many levels of dependency graph to traverse |
| `contextualDebugging.recentChangesDays` | 30 | How far back to check git history |

## Auto-Detection

On `init`, codebase-wiki detects:

1. **Workspace type**: monorepo (`.gitmodules`, `pnpm-workspace.yaml`, multiple `go.mod`) vs single-package
2. **Languages**: Go (`go.mod`), TypeScript (`package.json` + `tsconfig.json`), Kotlin (`build.gradle.kts`), Python (`pyproject.toml`), Rust (`Cargo.toml`), Java (`pom.xml`)
3. **Sub-packages**: each directory with a manifest file gets its own documentation
4. **Package manager**: bun, pnpm, yarn, npm, go mod, cargo, gradle
5. **Entry points**: `cmd/*/main.go`, `src/index.ts`, `main.py`, `src/main.rs`

## OpenCodeRAG Integration

If `.opencode/opencode-rag.json` exists, codebase-wiki auto-detects and reuses:

- `embedding.provider` and `embedding.model` — from the `description` section first, then `embedding`
- `baseUrl` and `apiKey` — for remote LLM providers
- `rag_db` path — reuses the existing vector store directory

This means a project with OpenCodeRAG already configured needs zero manual config for codebase-wiki.

## Environment Variables

| Variable | Description |
|----------|-------------|
| `WIKI_DB_PATH` | Override storage path (default: `.codebase-wiki/rag_db`) |

## Storage Format

Documentation and notes are stored as JSON files:

```
.codebase-wiki/rag_db/
├── docs.json    # [{ id, service_name, service_path, language, content, sections, indexed_at }]
└── notes.json   # [{ id, type, topic, content, context, tags, authored_by, authored_at }]
```

## CLI Options

```bash
codebase-wiki serve --port 3080    # Custom port for web UI
codebase-wiki search <query>       # CLI search (reads from disk)
codebase-wiki get <service>        # CLI retrieval
```
